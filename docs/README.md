# docs/

This directory is intended for **classic documentation** (user-facing docs, technical references, integration notes).

Planning / roadmap / status artifacts live in `plan/`.

Start at the repo entry point: `AGENTS.md`.

Human-friendly overview (what works today + ways ahead): `docs/OVERVIEW.md`.

Provider integration (via `req_llm`) + attachment safety config: `docs/PROVIDERS.md`.

Release/tag notes (what each semver tag contains): `docs/RELEASES.md`.
